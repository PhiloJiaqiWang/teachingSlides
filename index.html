Core Idea Slide 3

If this is true...

...then run this code.

if (condition) { doSomething(); }
  • The test goes inside ( )
  • The action goes inside { }
  • If the condition is false, the action does not happen
Think like a game

If the player touches a coin, then score goes up.

If the up key is pressed, then the player moves.

If score reaches 10, then level 2 starts.

Act It Out Slide 6

Physical coding activity

var clap = 0; function practice() { var spin = false; clap = clap + 1; if (clap == 4) { spin = true; clap = clap - 1; } }

Each click adds 1 clap. When clap reaches 4, the code turns on spinning and drops clap back to 3. That makes the pattern repeat.

Player Slide 8

Make the player move

  1. Create a function like playerMovement()
  2. Add an if statement
  3. Use keyDown("up") or another key
  4. Change sprite.x or sprite.y
  5. Call the function in the draw loop
if (keyDown("up")) { egg_1.y = egg_1.y - 4; } if (keyDown("right")) { egg_1.x = egg_1.x + 4; }

Rule to remember: up and left usually subtract, down and right usually add.

Enemies Slide 9

Make enemies move on their own

  • Use velocityX or velocityY
  • Enemies usually do not wait for player input
  • Create edge sprites to stop them leaving the screen
  • Use collisions for bouncing, blocking, or combat
fox.velocityY = 3; createEdgeSprites(); fox.collide(edges); if (fox.isTouching(egg)) { egg.destroy(); }

Question: should the enemy bounce, chase, patrol, or guard one area?

Score Slide 10

Track points

var score = 0; if (player.isTouching(coin)) { score = score + 1; }
  • Put the score variable in the header
  • Increase it when the player completes a goal
  • Common triggers: coins, jumps, hits, survival time
